# 07 · RaiOS Dashboard Target

RaiOS is an Agency Operating System for:
- finding potential website clients
- auditing websites
- generating demos
- sending outreach
- tracking follow-ups
- managing proposals/delivery

## Dashboard purpose

The RaiOS dashboard should answer:
- who should I contact today?
- why this lead?
- what problem was found?
- what demo/proposal is ready?
- what action should happen next?

## But target is visual, not only functional

A correct structure is not enough.

The dashboard must feel:
- clean like reference 10
- list/table precise like reference 05
- home/focus rich like reference 07
- product-specific like references 03/04
- typographically refined across all references
- not generic Codex dashboard

## Do not make literal copy

Do not use:
- purple just because reference 07 is purple
- map unless the product needs map/canvas
- black sidebar unless it improves the design
- SVG unless it is good or requested as asset

## Successful RaiOS dashboard

Should have:
- strong typography plan
- clean palette
- precise component system
- lead queue/list that feels premium
- right context rail if useful
- home/focus module with orientation
- enough density to feel complete
- custom enrichment only where it improves the UI

## Unsuccessful RaiOS dashboard

Fails if:
- it looks like previous Codex attempts
- it uses cream/grid aesthetic not grounded in references
- it is clean but visually poor
- it has fake previews and calls them credible
- it claims 8/10 because pieces are present
