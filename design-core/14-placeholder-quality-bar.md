# 14 · Placeholder Quality Bar

Placeholders are allowed only if they are refined and honest.

## Why

Codex often uses placeholders:
- initials
- fake browser lines
- colored boxes
- skeleton cards
- random pills

These usually make the UI look like Codex.

## High-quality placeholder rules

A placeholder must:
- fit the typography system
- fit the palette
- be subtle
- be consistent
- be visually quiet
- not pretend to be final custom design

## Identity markers

If real logos/favicons are missing:
- use neutral consistent initials
- avoid random pastel colors
- avoid rainbow identity systems
- use one or two accent systems maximum

## Website previews

If real screenshots are missing:
- either request assets
- or use a refined mini browser placeholder clearly as temporary
- do not use random skeleton rectangles
- do not overuse repeated fake previews

## SVG placeholders

If an SVG is temporary:
- make it abstract and refined
- no complex characters
- no childish shapes
- no stock SaaS illustration feeling

## Scoring

If placeholders dominate the visual impression, overall score cannot exceed 6/10.

If placeholders look generic Codex, overall score cannot exceed 5/10.
