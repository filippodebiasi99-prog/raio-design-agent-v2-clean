# 06 · Custom Enrichment

Custom elements enrich a strong foundation.

They do not replace typography, palette and component quality.

## Possible custom enrichment

- SVG illustration
- custom map
- branded chart
- custom timeline
- icon set
- website previews
- favicons/logos
- micro-pattern
- motion
- custom score visual

These are optional and contextual.

## Quality bar

A custom element must be:
- product-specific
- well integrated
- visually refined
- aligned with typography/palette/components
- not crude
- not filler

## If Codex cannot make it well

Do not fake it.

Request the asset with a brief:
- asset name
- priority
- purpose
- placement
- style direction
- what to avoid
- what the user should provide
- temporary fallback

## Wrong behavior

Do not add:
- random SVG because user likes SVG
- map because user liked a map reference
- black sidebar because user liked one sidebar
- logos everywhere because user liked entity-specific logos
- grid background to fake visual richness
- fake mini previews and call them final

## Custom score

If custom elements are weak, remove or replace with asset request.

If custom is used to hide weak foundation, score cannot exceed 5.
