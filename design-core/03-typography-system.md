# 03 · Typography System

Typography is the first visual layer.

A software screen is mostly text. If the typography is bad, the UI is bad.

## Font family

The user likes Montserrat, but V2 must not hardcode only Montserrat.

Good directions:
- clean geometric sans
- refined grotesk
- product UI sans with personality
- editorial sans when appropriate

Avoid:
- default-looking Inter
- uncalibrated system font
- heavy startup-template typography
- generic SaaS font treatment

If Inter is used, it must be deliberately refined.

## Size range

Suggested dashboard scale:
- page/home title: 22-30px
- main module title: 18-24px
- section title: 13-16px
- row title: 12-14px
- body/supporting text: 12-14px
- metadata/labels: 10-12px
- large numbers: 22-34px only when needed

Avoid:
- 40px+ hero headlines in functional dashboards
- huge bold marketing headlines inside product UI
- large type everywhere
- tiny unreadable metadata

## Weight system

Suggested:
- metadata: 400-500
- body: 400-500
- row titles: 500-600
- section titles: 550-650
- page titles: 600-700 max
- numbers: 600-700 max

Avoid:
- 700/800 everywhere
- bold labels everywhere
- heavy headings in every card
- weight used to compensate for poor layout

## Letter spacing

Prefer natural or slightly tight tracking.

Use:
- slight negative tracking for larger titles if it improves refinement
- neutral tracking for body
- subtle labels

Avoid:
- excessive positive tracking
- all caps with wide tracking everywhere
- default spacing if it looks loose or uncalibrated

## Line height

Use:
- headings: 0.95-1.15
- module titles: 1.05-1.2
- body copy: 1.35-1.55
- rows: compact but readable

Avoid loose generic SaaS line height or cramped multi-line text.

## Casing

Prefer sentence case and calm labels.

Use uppercase sparingly.

## Typography score

If typography feels default or generic, score cannot exceed 6.

If typography is bad and palette is bad, score cannot exceed 4.
