# 05 · Spacing and Components

Layout, spacing and components are the third visual layer.

## Spacing

Every spacing decision should be intentional:
- page margins
- topbar height
- section gaps
- card padding
- row height
- distance between title/body/meta
- rail width
- whitespace balance

Avoid:
- huge empty middle
- everything too close
- cramped sections
- oversized card padding
- random gaps
- dense chaos
- empty sparse wireframe feeling

## Component system

Define before coding:
- card surface
- card radius
- card padding
- button height/radius
- input/search height
- list row height
- pill size/radius
- border/shadow logic
- selected state
- icon size
- sidebar/topbar behavior

## Cards

Cards should share a system but not all have the same internal template.

Avoid:
- card soup
- card inside card inside card
- repeated title/subtitle/icon modules
- heavy borders everywhere

## Buttons

Buttons should be compact, refined and consistent.

Avoid:
- too square
- too bubbly
- too big
- too generic

## Lists/tables

For CRM/list-heavy UI, rows should feel premium:
- identity
- context
- score/status
- next action
- visual preview if needed
- subtle active state

Avoid:
- spreadsheet default
- chunky rows
- thick colored side strips
- malformed rings
- fake repeated previews

## Navigation

Navigation is not required to be unusual.

It can be simple if refined.

Avoid inventing a sidebar style that is not grounded in references.

## Component score

If component system feels generic, score cannot exceed 6.

If spacing is chaotic or too cramped, score cannot exceed 5.
