# 04 · Palette System

Palette is the second visual layer.

Bad colors ruin the first impression.

## Palette order

Define in order:
1. base background
2. primary surface
3. text
4. muted text
5. borders/separators
6. primary accent
7. status colors
8. selected states
9. optional decorative colors

Do not start with accent colors.

## Preferred feel

Palette should feel:
- clean
- controlled
- premium
- intentional
- not muddy
- not random
- not generic gray admin

## Avoid

Avoid:
- dirty cream / panna background
- muddy beige
- greenish-gray backgrounds
- forest green atmosphere
- medical/eco dashboard palette
- pastel orange/rust with green
- low contrast green/beige UI
- decorative grid backgrounds unless clearly justified by reference quality

## Accent logic

Green can be used as a controlled accent.

Do not use green as:
- whole atmosphere
- full dark sidebar
- global background wash
- proof of branding

## Status colors

Status colors must be restrained and consistent.

Avoid pastel soup.

## Palette score

If the palette fails at first glance, score cannot exceed 5.

If the palette is not traceable to the selected reference stack, reference fidelity is low.
